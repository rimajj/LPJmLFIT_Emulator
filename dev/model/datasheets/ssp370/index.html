Datasheet — SSP370 (MPI-ESM1-2-HR) OOD ground truth

A dataset datasheet (after Gebru et al., "Datasheets for Datasets") for the out-of-distribution warming ground-truth run. Provenance from DESIGN.md §4 and config/paths.yaml.

Motivation

The realistic out-of-distribution (OOD) trajectory used to test generalization under warming. It is a real GCM scenario, not a synthetic temperature delta — a synthetic delta would be exactly the factorial/stylized perturbation the design forbids (DEVELOPMENT_PLAN.md §3). This is the transient the sibling equilibrium-ML emulator failed to project, and therefore the sharpest test of the hybrid's dynamical layer (DESIGN.md §6).

Composition

  • Model. LPJmL-FIT v5.6.004, git b2e5ca9 — same binary and config family as the Historical run; only the forcing scenario differs.
  • Domain / grid. Same 0.5° grid, 67,420 cells (63,119 with tree data), npatch = 25.
  • Period. Transient 2020–2100, continued from the Historical seed1 restart_2019.lpj ("orderA"). Forcing spans 2015–2100.
  • Scenario / forcing. MPI-ESM1-2-HR, SSP370 (high emissions), regridded to the model grid; same five climate variables as Historical (tas, pr, lwnet, rsds, huss). Exact files in config/paths.yaml:lpjml.inputs.ssp370.
  • CO₂ — held CONSTANT after 2019 (global_co2_ann_1700_2019_const_2100.txt). This is the defining property: because LPJmL-FIT runs with_nitrogen="no", unbounded CO₂ fertilization would blow vegetation carbon up, so the future is run at constant CO₂ (ADR 0004).
  • Seeds. seed1 complete; seed2 in progress — the SSP noise-floor pair needed to bound the OOD distributional error (gate the OOD-distribution evaluation on its completion).

Sizes

  • ind CSV ≈ 180 GB (seed1) (81 years).
  • Raw GCM NetCDFs (config/paths.yaml:lpjml.inputs.ssp370_raw_gcm) hold 9 variables including sfcwind and (possibly) ps — the source for the wind + surface-pressure inputs the energy layer E needs and the LPJmL core omits (DESIGN.md §7.3; config/paths.yaml:lpjml.energy_extra_inputs).

Uses

  • The OOD stress test = warming + precipitation variability at (near-)constant CO₂, not rising CO₂ (DEVELOPMENT_PLAN.md §5). Check physical plausibility (realistic ET response, no runaway water loss) and that conservation holds under the transient.
  • Source of sfcwind/ps for component E.
  • Must not be used to project CO₂-fertilization responses — the constant-CO₂ regime forbids it (Limitations).

Collection process, distribution & maintenance

Generated on the PIK cluster via the SSP370 production run script (config/paths.yaml:lpjml.run_script_ssp370; --ntasks=2048, --qos=short, wall ~3.5 h). Internal PIK storage (config/paths.yaml:lpjml.ground_truth.ssp370_seed{1,2}); not committed to git, tracked via DVC. Confirm the .clm header is 365-band daily in Phase 1 (the residual concern is daily-weather realism, not resolution). Reproducibility ledger per Reproduce a result.
