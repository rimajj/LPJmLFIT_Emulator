# `scripts/`

Utility scripts for the `LPJmLFITEmulator` project. All are **Base-only** (no external
packages) so they run in the Phase-0, dependency-free package.

## `gen_diagrams.jl` — code-derived model/data-flow diagrams

Reads the model's **own** component/flux registry — `src/registry.jl`
(`COMPONENTS`, 4; `FLUXES`, 8) — and emits Mermaid text to `docs/src/generated/`:

| Output | What it is |
|---|---|
| `docs/src/generated/dataflow.mmd`   | `flowchart LR` of the ATM / S / F / E subsystems; edges labelled by **payload + timescale** (thick `==>` marks a **conserved** handoff). Nodes are styled per `kind` (ml / physics / hybrid / external) so the **ML node stands out**. |
| `docs/src/generated/components.mmd` | Component overview — one node per component with **kind + timescale** (+ description). |

Both files carry the header `%% AUTO-GENERATED by scripts/gen_diagrams.jl — DO NOT EDIT`.
Node counts are kept **subsystem-level** (4 nodes) per the Mermaid caveats in
`ENGINEERING_STANDARDS.md` §5 (Mermaid degrades past ~50–100 nodes).

### Run it

```bash
JULIA=/p/system/packages_rhel9/tools/julia/1.10.0/bin/julia
$JULIA --project=. scripts/gen_diagrams.jl          # (re)write the two .mmd files
$JULIA --project=. scripts/gen_diagrams.jl --check   # CI staleness gate (exit 1 if stale)
```

(`--project=.` from the repo root, or `--project=/p/projects/open/Jamir/esm_land_emulator`.)

### `--check` (CI)

`--check` regenerates the diagrams to a temp dir and byte-compares them against the
committed files. It **exits 0** if they match and **exits 1** (listing the stale files)
if they differ. This is the `git diff --exit-code -- docs/src/generated` equivalent and
is meant to run on every push:

```yaml
# .github/workflows: after buildpkg
- run: julia --project=. scripts/gen_diagrams.jl --check
```

If it fails, run without `--check`, review the diff, and commit the regenerated files.

## Governance: curated vs. derived diagrams (ENGINEERING_STANDARDS §5)

Two sets of diagrams exist on purpose, and they play different roles:

- **Derived** — `docs/src/generated/*.mmd`, produced by this script from the registry.
  These are the **source of truth**: always true because they are regenerated from the
  code, and they act as a **diff alarm**. If the registry (`src/registry.jl`) changes,
  these files change, and `--check` fails in CI until they are regenerated and committed.
  **Never hand-edit them** — edits are overwritten on the next run.

- **Curated** — `docs/src/assets/diagrams/*.mmd` (`components.mmd`, `coupling.mmd`,
  `dataflow.mmd`), **hand-authored**. These are the **owner's mental model**: small,
  polished, readable, and free to add framing the registry does not encode (the shared
  state, E's bounded NN, the Daily/Annual timescale tiers, the conserved budgets).

**The rule that resolves the tension:** when a derived diagram changes (the diff alarm
fires), that is the signal to **update the matching curated diagram by hand**. The derived
set keeps everyone honest; the curated set keeps everyone oriented.

All diagrams are committed as **Mermaid text only** (no rendered SVG) so diffs stay
reviewable, and all use `flowchart`/`graph` (never C4) so they render natively on GitHub
and in the Documenter site (via DocumenterMermaid.jl).
